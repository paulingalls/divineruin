# Phase 5 Audit — Crafting M5.1 + M5.2 (Recipe/Material + Workspace/Resolution)

Sprint-004 / Milestone 4 (story-001). Read-only audit of `docs/milestones/05_crafting.md` §M5.1 + §M5.2 against `docs/game_mechanics/game_mechanics_crafting.md` (587 lines, §Crafting Resolution / §Recipe System / §Workspace Access / §Materials) and shipped code in `apps/agent/`, `apps/server/`, `packages/shared/src/entities/`, `content/`, and `scripts/migrations/`. Status legend (per story-001 AC): **BUILT** = code present and matches spec; **DESIGNED** = spec is well-defined and partial code exists, but the symbol/feature is missing, renamed, or diverges from spec (corresponds to sprint-002/003 `aspirational` + `unverified`); **NOT_SHIPPED** = no implementation found at all.

**Headline finding:** The full M5.1+M5.2 crafting pipeline (recipe slots, three-track acquisition, four workspace types, three-check resolution) is unshipped. What exists is a thin prototype: a server-side `CRAFTING_RECIPES` map with 4 hard-coded recipes (`iron_sword`, `healing_poultice`, `ward_stone`, `reinforced_shield`) at `apps/server/src/activity_templates.ts:94+`, a minimal `CraftingRecipe` interface (id, name, duration, materials as string[], result, skill, dc, npc_id — none of category/tier/workspace_required/study_cost/discovery_sources/narration_cues), and an `async_rules.resolve_crafting()` outcome resolver at `apps/agent/async_rules.py:34-129` whose outcome tiers (`success`/`partial`/`unexpected`/`failure`) diverge from the spec's four-tier model (`Exceptional`/`Success`/`Partial`/`Failure`) and which has no pre-flight recipe-knowledge / skill-tier / workspace check. Materials are tracked as bare item-id string-arrays — no `MaterialReq` shape, no tier minimum, no substitutable flag, no materials catalog. Zero crafting-related DB migrations exist (migrations 001-017 inventoried; no `recipes`, `recipe_slots`, `player_known_recipes`, `materials_catalog`, or `workspace_rentals` table). No `content/recipes.json`; `content/items.json` contains only 1 entry typed `material` (`hollow_bone_fragment`). The capstone should uncheck nearly every acceptance box across M5.1 and M5.2.
