## M1.5 — Hybrid Counter Claim

The acceptance criterion "skill practice training increments the same use counter as session use (M1.2 hybrid advancement)" requires a single counter mutated by both code paths.

- **Session-use path**: `apps/agent/check_tools.py:190` calls `check_resolution.record_skill_use(tiers, skill_lower, {skill_lower: skill_adv["use_counter"]}, ...)` from inside the `request_skill_check` agent tool. Result is persisted via `db_mutations.update_skill_advancement` (`apps/agent/db_mutations.py:85` writes to `skill_advancement` table).
- **Async-training path**: `apps/agent/async_worker.py:300` calls `check_resolution.record_skill_use(tiers, training_skill, counters, narrative_moment=narrative)` inside `advance_training_cycles` when `activity_type == "skill_practice"`. Result is persisted via the same `db_mutations.update_skill_advancement` (`async_worker.py:309`).
- Both paths read prior state from `db_queries.get_single_skill_advancement` and write back to the single `skill_advancement` row keyed by `(player_id, skill_id)`.

**Status: confirmed.** The counter is genuinely shared. Per-completion increment is +2 for the "fundamentals" decision and +1 for "advanced" (`training_rules.py:247`).
