Status legend (per story-003 AC): **BUILT** = code present and matches spec; **DESIGNED** = spec is well-defined and partial code exists, but the symbol/feature is missing, renamed, or diverges from spec; **NOT_SHIPPED** = no implementation found at all.

**Headline finding:** The durability system (4 tiers, hit-depletion, Hollow double-corrosion, repair pricing) is entirely **unshipped**. No `Durability` enum, no `apply_durability_damage` / `check_item_condition` / `calculate_repair_cost` functions, no `durability` column on items, no per-item HP tracking. The `Item` interface at `packages/shared/src/entities/item.ts:9-25` carries 13 fields (id, name, tier 1|2, type, subtype, rarity, description, tags, weight, effects, value_base, value_modifiers, lore, found_in) — **missing** durability, damage_dice, AC, properties, price (spec uses `value_base` but spec's catalog values are in sp/gc not the abstract `value_base` integer), magic-item-tier-gating fields. `content/items.json` contains 29 items with NO `durability` field (`sum(1 for i in items if 'durability' in i)` → 0). The shipped item catalog covers ~5% of the spec catalog: 7 weapons (vs ~15 spec weapons + 6 magic-item weapons), 5 armor (vs ~10 spec armor + magic armor), 6 consumables (vs spec's ~15 potions/poisons/ammunition), 1 material (vs ~40 spec materials — also noted in M5.1 audit), 4 quest items + 3 accessories + 3 equipment + 0 tools (vs ~15 spec adventuring gear + magic tools). The `Item.tier: 1 | 2` field is a 2-value union — spec defines 4 tiers (Common/Uncommon/Rare/Legendary mapped to tiers 1-4). No `items_catalog` migration exists; the `Item` TS interface is the de facto schema.

Naming caveats (deliverables only — all are NOT_SHIPPED unless tagged otherwise):
- Spec deliverable `apply_durability_damage(item_state, damage_amount, is_hollow_zone)` pure function — NOT_SHIPPED. `grep -r 'apply_durability_damage' apps/ packages/` → 0 matches.
- Spec deliverable `check_item_condition(item_state)` pure function — NOT_SHIPPED.
- Spec deliverable `calculate_repair_cost(item_state, durability_tier)` pure function — NOT_SHIPPED.
- Spec deliverable `items_catalog` migration with durability + full type fields — NOT_SHIPPED. The shipped `Item` interface at `packages/shared/src/entities/item.ts:9-25` is incomplete vs spec.
- Spec deliverable `content/items.json` expanded with all item types, stats, and crafting-system properties — DESIGNED. File exists with 29 items but missing nearly all crafting-system properties (no damage_dice, AC, durability, properties, magic-tier).

## Material gaps

Content / data gaps that block M5.4 from leaving DESIGNED state:

1. **`Item.durability_tier` + `Item.current_hits` fields** — do not exist. `Item` interface at `packages/shared/src/entities/item.ts:9-25` has 13 fields, none durability-related. Adding them requires a schema migration AND a sweep across all 29 items.json entries to backfill (which would be a content-authoring task — implementer must decide each item's spec-default durability tier).
2. **`Item.damage_dice` + `Item.properties` for weapons** — do not exist. Weapon damage lives in prose `description` and unstructured `effects[].description`. Spec's Weapons Standard Catalog (crafting.md:357-374) and Properties (crafting.md:378-388) need to be authored as structured data, 15+ entries.
3. **`Item.ac` + `Item.armor_properties` for armor** — do not exist. Armor AC + STR-requirement + Stealth-disadvantage need structured fields. Spec's Armor Standard Catalog (crafting.md:392-402) is 10 entries.
4. **Adventuring Gear sub-category** — 19 spec entries (Backpack to Arcane Focus, crafting.md:406-424). Cross-doc deps with M3 Magic (Holy Symbol, Arcane Focus required for casting) and M5.1 (Crystal Flask required for potion crafting). NEW vs M5.4 milestone deliverables (milestone lists tools, not gear); implementer should split tools / adventuring-gear / consumables cleanly.
5. **Magic-item content** — 6 spec Rare + 4 spec Legendary unique items (crafting.md:430-515). Each carries: crafting-recipe ref (cross-ref to M5.1), audio cue (cross-ref to audio-first invariant in CLAUDE.md), attunement requirement (NEW field), per-item drawback (e.g., Stillheart's "Stay. Rest." offer). All absent from items.json.
6. **`items_catalog` table** — no DB persistence. Items are loaded from `content/items.json` at server start (presumed; not verified) but not queryable by category/rarity/tier from the agent without scanning the JSON. Spec deliverable text says "DB migration updates `items_catalog` table with durability fields" — implementer must decide whether items move to DB or stay as JSON.

## Out-of-scope findings (Sprint-spec-cleanup punch list)

- **Repair-pricing axis conflict.** Spec at crafting.md:542-549 keys repair cost on item-rarity-tier (Common/Uncommon/Rare/Legendary). Milestone acceptance bullet 4 says "scales with durability tier and current damage level". These are 2 (rarity vs durability) or 3 (+ damage-level) different axes. Capstone should record for customer resolution — either spec or milestone wording needs to win.
- **Thornridge's Stand exception.** Legendary item Thornridge's Stand (crafting.md:516) carries "Cannot be crafted" — quest-only acquisition. Milestone acceptance bullet 9 assumes all Magic items are gated by crafting tier; this exception breaks that gate. Add a `quest_only: bool` field OR record a documented exception list.
- **`Item.tier: 1 | 2` is undersized.** TS type at `item.ts:13` accepts only `1 | 2`; spec defines 4 rarity tiers (1=Common through 4=Legendary at crafting.md:347-353). M5.4 needs to widen this to `1 | 2 | 3 | 4`. Mechanical breakage when a future Rare-tier item lands with `tier: 3`.
- **Audio-cue field missing from `Item` interface.** Spec magic items carry explicit "Audio:" descriptions (e.g., Blade of the Ashmark soft radiant hum). CLAUDE.md audio-first invariant should be enforced at the schema layer. Recommend adding `audio_cue?: string` to `Item` interface.
- **Repair-tier-vs-skill-tier coupling.** Spec at crafting.md:524-530 pairs each durability tier with a repair-skill-tier requirement (Fragile→Untrained, Standard→Trained, Reinforced→Expert, Masterwork→Master). The acceptance bullet on repair cost doesn't mention this. Note for whoever ships M5.4: the repair flow needs to gate on the player's Crafting skill tier matching the item's durability tier.
